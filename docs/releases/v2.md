License: MIT. The plugin is free to use, copy, modify, and distribute under
the terms in `LICENSE`.

## Artifact Authority

`REQUIREMENTS.toml` is the release-source authority. It owns required
artifacts, executable scripts, stale path denials, version files, and
CI-tracked governance files. `scripts/status.sh --release-source` and
`release-check all` consume that manifest rather than a hand-maintained list in
this document.

Required release-source files must be native files, not symlinks. New Rust
sources, tests, Nix files, NixOS modules, hooks, templates, Claude skills, and
runtime assets are release failures until they are declared in
`REQUIREMENTS.toml`.

## Breaking Changes

- `--overlay` is removed. Existing repositories use `--merge`.
- Merge removes stale legacy governance files instead of preserving them.
- Consumer repositories use the plugin-owned `.codex/scripts/task-registry` wrapper.
- Compatibility shims for old hook paths and settings files are not supported.
- New Task Manifests require `schema_version = 2` and typed behavior
  verifiers. Completed legacy v1 manifests remain historical evidence only.
- Runtime governance writes are task-bound, with `docs/plans/*.md` plan
  bootstrap as the narrow exception.
- Runtime checks emit schema-backed diagnostics for release source, source
  limits, mutation scopes, behavior verifiers, and installer dry-runs.
- Completed and cancelled registry tasks are terminal; changed follow-up work
  requires a new task id.
- Receipt events use schema version 2 hash chains; malformed, unchained, or
  tampered receipts block production readiness.

## Migration Notes

1. Vendor or submodule this plugin under `plugins/agent-governance`.
2. Run `plugins/agent-governance/scripts/install-to-workspace.sh --config project.config.toml --merge`.
3. Commit every path listed by `REQUIREMENTS.toml`.
4. Run `.codex/scripts/task-registry validate`, `.codex/scripts/task-registry source-limit check`, and `plugins/agent-governance/scripts/status.sh --strict`.

## Audit Policy

Production v2 release requires:

- `cargo tree --locked --manifest-path rust/task-registry-flow-cli/Cargo.toml -d`
- `cargo run --locked --quiet --manifest-path rust/task-registry-flow-cli/Cargo.toml -- release-check all`
- `cargo test --locked --manifest-path rust/task-registry-flow-cli/Cargo.toml`
- `cargo clippy --locked --manifest-path rust/task-registry-flow-cli/Cargo.toml -- -D warnings`
- `cargo audit`
- `cargo deny --locked check --config rust/task-registry-flow-cli/deny.toml`
- `nix flake check --no-build --all-systems`

Missing `cargo-audit` or `cargo-deny` blocks release unless a governed waiver sets `AGENT_GOVERNANCE_ALLOW_AUDIT_TOOL_WAIVER=1`.

`scripts/release-audit.sh` is cwd-independent: it resolves the git repository root, changes there before running source-limit and schema-backed release checks, and then runs Cargo audit checks against `rust/task-registry-flow-cli`.

## Final Gate

Run from a clean tree:

```bash
scripts/status.sh --release-source
scripts/release-audit.sh
bash scripts/test-install-modes.sh
bash scripts/test-release-readiness.sh all
nix flake check --no-build --all-systems
cargo run --quiet --manifest-path rust/task-registry-flow-cli/Cargo.toml -- validate
agy plugin validate .
git status --short
```

The final `git status --short` output must be empty.
Set `AGENT_GOVERNANCE_FINAL_RELEASE=1` for final release validation; local
waiver flags are rejected in that mode.
